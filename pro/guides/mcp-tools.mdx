Rasa MCP Tools is a local [Model Context Protocol (MCP)](https://modelcontextprotocol.io/) server that gives IDE copilots deep, Rasa-specific knowledge about your project. Instead of copying and pasting YAML into a chat window, you can ask your copilot to inspect your flows, add a new feature, validate your configuration, train a model, or talk to the running assistant — all in plain language, directly from your editor.

Rasa MCP Tools is included in **Rasa Pro 3.16 and later**. There is no separate package to install.

## What you can do

| Capability | Example prompt |
|---|---|
| **Inspect** | "What flows and slots are in this project?" |
| **Build** | "Add a new flow for flight bookings." |
| **Validate** | "Validate my project and explain any errors." |
| **Train** | "Train the model and tell me if it succeeded." |
| **Test** | "Send 'I want to transfer money' to the assistant." |
| **Debug** | "Get the assistant logs and explain why the flow was cancelled." |
| **Learn** | "Search the Rasa docs for how slot validation works." |

The MCP server exposes tools across six groups: documentation search, project introspection, schema retrieval, build and validation, runtime testing, and simulation and evaluation.

## How it works

Your IDE starts `rasa tools run`, a lightweight local process that speaks MCP over stdio and reads your project files directly. No cloud service is involved for most tools. The optional `talk_to_assistant`, `get_assistant_logs`, and `evaluate_agent` tools require a running Rasa server, which you start separately.

```
Your machine

  ┌──────────────┐  MCP (stdio)   ┌──────────────────────┐
  │ IDE Copilot  │ ◄────────────► │ Rasa MCP Server      │
  │              │                │ rasa tools run       │
  │ (your IDE)   │                │ reads project files  │
  └──────────────┘                └──────────┬───────────┘
                                             │ optional HTTP
                                             ▼
                                  ┌──────────────────────┐
                                  │ Rasa Server          │
                                  │ rasa run --inspect   │
                                  └──────────────────────┘
```

## Quick setup (recommended)

<Steps>
  <Step title="Open your project root in a terminal">
    Navigate to the root directory of your Rasa project — the folder that contains `config.yml`, `domain.yml`, and `data/`.
  </Step>
  <Step title="Run the setup wizard">
    ```bash
    rasa tools init
    ```

    The wizard:
    - Creates `.rasa/tools.yaml` with your server configuration
    - Downloads offline Rasa documentation into `.rasa/`
    - Generates MCP configuration for your IDE
    - Installs Rasa-specific agent skills for your project

    To accept all defaults without prompts, add `-y`:

    ```bash
    rasa tools init -y
    ```
  </Step>
  <Step title="Open your IDE from the project root">
    After the wizard finishes, open your IDE from the same project root directory and check that `rasa-tools` appears in your IDE's MCP server list.
  </Step>
</Steps>

## Manual setup

If the wizard did not configure your IDE, or you prefer to set things up by hand, follow the instructions for your IDE below.

<Tabs>
  <Tab title="Cursor">
    <Steps>
      <Step title="Create the MCP config file">
        Create `.cursor/mcp.json` in your project root with the following content:

        ```json
        {
          "mcpServers": {
            "rasa-tools": {
              "command": "rasa",
              "args": ["tools", "run", "--mode", "stdio"]
            }
          }
        }
        ```
      </Step>
      <Step title="Ensure rasa is on your PATH">
        Use the same `rasa` binary you use in a terminal where `rasa train` works. If you use a virtual environment, make sure it is activated, or use the full path to the binary.
      </Step>
      <Step title="Restart Cursor or reload MCP">
        Open **Settings → Tools & MCP** and toggle the `rasa-tools` server on. Confirm it shows as connected.
      </Step>
    </Steps>
  </Tab>
  <Tab title="VS Code">
    <Steps>
      <Step title="Create the MCP config file">
        Create `.vscode/mcp.json` in your project root:

        ```json
        {
          "servers": {
            "rasa-tools": {
              "type": "stdio",
              "command": "rasa",
              "args": ["tools", "run", "--mode", "stdio"]
            }
          }
        }
        ```
      </Step>
      <Step title="Reload VS Code">
        Reload the window or restart VS Code. Run the **MCP: List Servers** command from the command palette to confirm `rasa-tools` is listed and active.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Claude Code">
    Claude Code launches non-interactive shells and may not inherit your terminal's `PATH`. Use the **full path** to your `rasa` binary (run `which rasa` inside your activated virtual environment to find it).

    <Steps>
      <Step title="Add the server via the CLI">
        From your project root, run:

        ```bash
        claude mcp add rasa-tools -- /path/to/.venv/bin/rasa tools run --mode stdio
        ```

        To store the config in your project (so it can be shared via Git), add `-s project`:

        ```bash
        claude mcp add -s project rasa-tools -- /path/to/.venv/bin/rasa tools run --mode stdio
        ```
      </Step>
      <Step title="Verify the connection">
        Run `/mcp` in Claude Code and confirm `rasa-tools` shows as **Connected**.
      </Step>
    </Steps>

    Alternatively, create `.mcp.json` manually in your project root:

    ```json
    {
      "mcpServers": {
        "rasa-tools": {
          "command": "/path/to/your/.venv/bin/rasa",
          "args": ["tools", "run", "--mode", "stdio"]
        }
      }
    }
    ```
  </Tab>
  <Tab title="JetBrains">
    These steps apply to IntelliJ IDEA, PyCharm, WebStorm, and other JetBrains IDEs.

    <Steps>
      <Step title="Open MCP settings">
        Go to **Settings → Tools → AI Assistant → Model Context Protocol (MCP)**.
      </Step>
      <Step title="Add a new STDIO server">
        Click **Add** and choose **STDIO**. Enter the following JSON config:

        ```json
        {
          "mcpServers": {
            "rasa-tools": {
              "command": "rasa",
              "args": ["tools", "run", "--mode", "stdio"]
            }
          }
        }
        ```
      </Step>
      <Step title="Set the working directory">
        Set the **Working directory** field to your Rasa project root. This is required for the MCP server to find your project files.
      </Step>
      <Step title="Save and restart">
        Click **OK** and restart the AI Assistant. If `rasa` is not on your global `PATH`, point `command` at your virtual environment's Python binary instead:

        ```json
        {
          "mcpServers": {
            "rasa-tools": {
              "command": "/path/to/project/.venv/bin/python",
              "args": ["-m", "rasa", "tools", "run", "--mode", "stdio"],
              "env": {
                "RASA_LICENSE": "your_license_key_here"
              }
            }
          }
        }
        ```
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Runtime testing (optional)

The `talk_to_assistant`, `get_assistant_logs`, and `evaluate_agent` tools require a running Rasa server. Start it before using those tools:

```bash
rasa run --inspect
```

The `--inspect` flag enables the REST API automatically. Once the server is running, you can use prompts like:

```
Talk to the assistant: "hello", "I want to send money to Jen", "$50", "yes".
Check if the transfer_money flow completed.
```

```
Get the assistant logs and explain why the flow was cancelled.
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="rasa: command not found">
    Your IDE cannot find the `rasa` binary. Make sure `rasa` is on your `PATH`, or use the full path to the binary inside your virtual environment. Run `which rasa` in an activated terminal to find the correct path, then update the `command` field in your MCP config accordingly.
  </Accordion>
  <Accordion title="RASA_LICENSE not found or license error">
    Your IDE may not inherit environment variables from your interactive shell — this is common when launching from a Dock, Spotlight, or application menu rather than a terminal. Add the license directly to your MCP server config:

    ```json
    "env": { "RASA_LICENSE": "your-license-key" }
    ```

    On zsh, you can also export the variable in `~/.zshenv` (not `~/.zshrc`), which is read by all shell types including non-interactive ones:

    ```bash
    echo 'export RASA_LICENSE="your-license-key"' >> ~/.zshenv
    ```
  </Accordion>
  <Accordion title="Tool calls are slow on the first use">
    This is expected behaviour. The MCP server lazy-loads Rasa modules the first time a tool is called. Subsequent calls in the same session are significantly faster.
  </Accordion>
  <Accordion title="talk_to_assistant fails or times out">
    This tool requires a running Rasa server. Start it with `rasa run` or `rasa run --inspect` before invoking any runtime tools. The MCP server connects to `http://localhost:5005` by default.
  </Accordion>
  <Accordion title="No tools appear after connecting">
    Restart the MCP server process from your IDE. In Cursor, toggle the server off and on in **Settings → Tools & MCP**. In VS Code, run **MCP: List Servers** and choose restart. In Claude Code, run `/mcp` and restart. Then verify the tool list before retrying.
  </Accordion>
  <Accordion title="IDE says 'no MCP servers found'">
    Check that your MCP config file exists in the correct location and contains valid JSON. Expected locations: Cursor → `.cursor/mcp.json`, VS Code → `.vscode/mcp.json`, Claude Code → `.mcp.json` or `~/.claude.json`. Reload or restart MCP in your IDE after making changes.
  </Accordion>
</AccordionGroup>
