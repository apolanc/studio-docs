## Rasa Orchestrator has three roles

## Rasa Orchestrator

## has three roles

Directing control

“What should happen next?”

Choose between flows, tools (MCP), agents (A2A), knowledge retrieval (RAG), or fallback.

Maintaining state

“What do we know so far?”

Track conversation, slots, tool outcomes, pending goals, and active tasks.

Framing context

“What does this interaction mean right now?”

Understand modality (voice, chat), intent ambiguity, user status, channel, and system constraints.
