This page tracks new features, improvements, and bug fixes for Rasa Studio across all releases. Entries are ordered from most recent to oldest. For changes to the underlying Rasa framework and Rasa Pro, see the [Rasa Pro changelog](https://www.rasa.com/docs/rasa/changelog).

<Update label="Spring 2026" description="Released April 2026">
  **Analytics Dashboard**

  Studio now ships a built-in analytics dashboard. No external setup required.

  - **Conversation Volume** — total sessions, split into successful and unsuccessful based on configurable success criteria
  - **Containment** — conversations handled end-to-end without escalation
  - **Latency** — average time from user message to assistant reply
  - **CSAT** — collected automatically via the `pattern_customer_satisfaction` system flow

  Each metric includes a month-over-month comparison. Containment insights surface the user journeys with the lowest rates so you know where to focus. Latency insights show the breakdown over time.

  **Conversation Review Filters**

  New filters for **channel**, **language**, and **tag** in Conversation Review. Filters are additive — stack all three or use one. Reset clears everything at once. Conversations that previously required manual scrolling to find are now a few clicks away.

  **Built-in CSAT Collection**

  The new `pattern_customer_satisfaction` system flow triggers automatically after `pattern_completed` when the user indicates they are done. It collects a `csat_score` slot with two values (satisfied or unsatisfied), responds accordingly, and asks only once per session. The pattern, its responses, and its trigger conditions are all customizable.

  If you already have a custom CSAT flow, populate the standard `csat_score` slot with the allowed values (`satisfied`, `unsatisfied`, `skipped`). Analytics picks it up without any changes to your user experience.
</Update>
