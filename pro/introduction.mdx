Rasa Pro is the pro-code interface of the Rasa platform. It gives developers and engineers direct control over every layer of an AI agent — from the dialogue system and business logic to integrations, LLM configuration, and deployment infrastructure. If you want to write flows in YAML, wire up custom Python actions, route across multiple models, or run entirely on-premise, Rasa Pro is where you do it.

Rasa Pro is built on [CALM (Conversational AI with Language Models)](/pro/concepts/calm), a dialogue system that combines structured flows with LLM understanding. CALM lets the language model interpret user input while your hand-authored flows enforce business logic — giving you the fluency of a modern LLM agent without sacrificing reliability or auditability.

<Note>
  You may see the terms "Rasa" and "Rasa Pro" used interchangeably in the documentation. "Rasa Pro" specifically refers to the pro-code, developer-facing part of the platform. [Rasa Studio](/studio/what-is-studio) is the companion no-code interface for business users and conversation designers.
</Note>

## Developer Edition License

The Rasa Developer Edition is **free** and lets you build and run Rasa agents locally and in production. A Developer Edition license covers:

- Up to **1,000 conversations per month** for customer-facing agents
- Up to **100 conversations per month** for internal employee-facing agents

[Request your free Developer Edition license key →](https://rasa.com/rasa-pro-developer-edition-license-key-request/)

Once you have a key, set it as an environment variable before running any Rasa command:

```bash
# Linux / macOS
export RASA_LICENSE="your-rasa-pro-license-key"

# Windows
setx RASA_LICENSE your-rasa-pro-license-key
```

## Installation Options

Choose the path that fits your workflow:

| Option | Best for |
|--------|----------|
| **Browser playground** — [hello.rasa.com](https://hello.rasa.com/) | Prototype an agent in minutes with no setup |
| **Developer Quickstart** (pip + MCP tools) | Developers who want a guided first-run experience |
| **Python / pip** | Developers comfortable with Python and virtualenvs |
| **Docker** | Teams that prefer a containerized, reproducible environment |

See the [Quickstart](/pro/quickstart) for step-by-step instructions for each option.
